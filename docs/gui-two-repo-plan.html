🖥️ GUI 配合兩 Repo 架構改造計劃

把「建 Repo B、放引擎、設金鑰、跑、切 live」全變成按鈕——讓只懂股票的人也能操作 · 2026-06-04

狀態：設計提案目標：使用者只填金鑰向後相容舊 fork 模式測試矩陣 v2（完整）
1 背景2 關鍵衝擊 ★3 現況→目標4 核心改動 5 逐檔影響6 引擎端配合7 測試矩陣 ★完整8 分階段+發版

1. 背景

GUI（tradingadmin）目前綁舊 fork 模型：精靈幫你 Fork tech-rebalance（整包引擎碼複製進你的 repo）。 但我們已驗證兩 repo 模型：使用者只有一個薄殼 Repo B（accounts.json + data/ + 薄 workflow + vendored 引擎 wheel），引擎按版本安裝。 GUI 要跟上，否則使用者得手動在 GitHub 點來點去。

2. 關鍵衝擊：Repo B 不含 strategies/schemas/brokers ★

兩 repo 模型下，策略 / schema / 券商定義都在引擎 wheel 裡，不在使用者 repo。 GUI 現在有幾處直接讀使用者 repo 的這些目錄，在 Repo B 會讀到空 → 必須改資料來源。
GUI 功能現在讀哪Repo B 會怎樣改成
策略下拉（catalog.list_strategies）repo 的 strategies/空讀引擎 manifest（§6）
券商下拉（catalog.list_brokers）repo 的 brokers/空讀引擎 manifest
必填 Secret 推導brokers/{id}.json讀不到manifest 內附 broker auth
schema 漂移橫幅（compat）repo 的 schemas/空讀 vendor/ wheel 版本
帳戶狀態（state_reader）repo 的 data/{id}/✅ 有不變
accounts.json 編輯repo 的 accounts.json✅ 有不變

3. 現況 → 目標

功能現在（fork）目標（兩 repo）
建立系統一鍵 Fork建 Repo B（建 repo + 範本 + vendored wheel）
取得修復同步上游（merge-upstream）更新引擎版本（換 wheel + bump daily.yml）
設金鑰per-account 金鑰不變（ACC1_*，免 token）
執行（靠排程）按鈕：dry-run / force / 看 log
排程編 daily_all_accounts.yml開關 Repo B daily.yml cron
paper/live（無）environment 切換 + live 防呆

4. 核心改動（摘要）

  • 4.1 精靈「建立 Repo B」：建 private repo + 推範本（含巢狀 daily.yml）+ 放 wheel；初始手動觸發、dry_run=true。
  • 4.2 引擎版本管理：顯示釘的版本；「更新引擎」列 Release → 下載 wheel → 換 vendor + bump daily.yml。
  • 4.3 執行按鈕：dry-run / 強制再平衡（live 跳確認）/ 看 runs+log。
  • 4.4 排程開關：Repo B daily.yml cron 啟用/停用。
  • 4.5 paper/live 切換 + 防呆：live 觸發交易需打字確認；可管理 paper/live 兩個 slug。
  • 4.6 drift 橫幅：改讀 vendor wheel 版本 vs App 支援範圍。
